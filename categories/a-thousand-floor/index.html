hdu4862 Jump

多校的第一天。题意是给一个n*m的矩阵，每个点上有权值，记maze。现在可以至多有出发k次的机会。每次可以从(i,j)向正右方的点跳跃，也可以向正下方的点跳跃。花费为|dx|+|dy|-1,如果两个点上的权值相同，那么可以获得收益S=maze[i][j].

构图方法是。可以先求权值为负的最小费用流。然后输出的时候取反即可。

1.超级源ss -> 源s 容量k,费用0

2.对n*m拆点为X,Y两部

源s -> X 容量1,费用0

Y -> 汇t 容量1,费用0

X -> Y 容量1,费用MOD

3.源s -> 汇t 容量k，费用0 这条边用于补充剩余的步数。

这里的MOD是一个恰好的较大数，一般为10^4到10^6之间。目的是既不压到下面影响其他流量，又不会爆int上限。这样一来就可以承担一个大的权重，从来保证每个点都被走到。

此时有。

mincost = m*n * MOD + |dx|+|dy|-1 - S

cost/MOD = m*n + (|dx|+|dy|-1 - S)/MOD
= m*n + 0

cost%MOD = 0 + (|dx|+|dy|-1 - S)%MOD
= |dx|+|dy|-1 - S

因此，当cost/MOD = m*n 的时候有解，所获收益为cost%MOD.

poj1324 Holedox Moving

好了。被虐一下午的BOSS.至此搜索专题基本毕业o(╯□╰)o
贪吃蛇问题，给出蛇的初始位置和障碍物，问蛇能否到达(1,1)
是一道状态设计很巧妙的题目。不过这里被STL卡了好久。换成手写队列后就一遍过了。
首先。我们考虑最极端的情况，蛇的长度最大的时候有8段，我们如果要记录这8段的位置，位置的可能是 20 * 20＝400，那么就可能会用到(400)^8的空间。

分析下去，我们会发现，如果蛇头的位置确定，那么蛇身可以从蛇头按４个方向一直走到蛇尾。这样状态数只为 20 * 20 * (4)^7 = 6553600
一个的数组还是可以承受的。这样就可以做到了。

poj3460 Booksort

这道题是刘汝佳黑书P169的例题。A*搜索。她的估价函数设计很有趣。
因为剪切是以一些连续段落为一个整体，这启发我们应该设计一个和“相对位置”有关系的h函数，例如h(s)=”后继段落正确的段落数目”。显然假设把连续段落S从P1的后面移动到P2的后面。那么只有3个段落的后继有变化:
S的最后一个段落,P1,P2.因此h每次最多减少3.虽然这时候仍不相容，但已经是常数级别。可以把h’=h/3作为估价函数值，则每次h最多减少1。但为了方便，可以把f扩大3倍，即f(s)=h(s)+3g(s)，这时候f仍然单调增。

PS: h(s)函数相容的定义是：h(s)<=h*(s),h(s1)<=h(s2)+c(s1,s2).
其中c(s1,s2)指状态S1转移到S2的费用。也就是状态转移时，下界h的减少值最多等于状态转移的实际代价。(h不能减少的太快)
